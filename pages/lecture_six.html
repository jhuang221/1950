Introducing CSS3

CSS3 promises to provide much greater control over the 'experience layer' of a user's interaction with a web page, and also to address some of the problematic layout issues of CSS2.

Technically, the current W3C recommendation for web developers and browser vendors is CSS2. However, if close attention is payed to browser support, many CSS3 features can be effectively deployed today.

CSS3 Module Development

CSS3 is being developed as a series of 'modules'. This allows certain features to be added as a sort of ongoing evolution of CSS, instead of needing to release a monolithic new version.

Since each module can be developed independent of the others, segments of the specification may progress faster (or slower) than others. Browser support for particular CSS3 modules can be very selective, as some of them are not near enough to completion. Other modules of CSS3 are closer to being settled, and several of these are implemented in many modern browsers.

In this course we will focus most attention on the features that are currently well supported by the major modern browsers.

  • W3C CSS3 Working Draft
  • CSS3 Browser Compatibility Test

Compatibility: when to apply CSS3

Some aspects of the user's visual experience should be considered critical and others non-critical. Since several CSS3 techniques are not well supported at present, only use CSS3 on things non-critical.

Critical (Not Ideal For CSS3)

  • layout
  • branding
  • usability
  • accessibility

Non Critical (Ideal For CSS3)

  • interaction (hover, active, focus, etc)
  • visual enhancements that might result from interaction
  • anything that 'adds value' but is not a crucial part of the core experience

Follow Design Priciples

Keep in mind: it is important not to get too carried away with the over-use of any particular technique. When a new, fancy feature is introduced, especially features that add 'visual flair', the tendency for some developers can be to use it everywhere. Judicious and subtle application can often be more effective than going overboard. For example, dont apply the text-shadow to all text on a page. Apply it to add contrast and draw attention to special points of interest. Remember the design principles!

Compatibility: vendor prefixes

Several CSS3 features are ready to go in most modern browsers, and can be used immediately.

Other features may not enjoy complete support yet, though browser vendors will sometimes implement experimental support for a CSS3 feature. To ensure a CSS3 technique is correctly deployed, vendor prefixes may need to be added to some property names.

Vendor prefixes

  • Firefox: -moz-
  • Microsoft: -ms-
  • Safari: -webkit-
  • Opera: -o-
  • Chrome: -chrome- * not usually required, as the Chrome kernal is the same as Safari: -webkit-

In order to apply a vendor prefix to a CSS3 property such as transition, add the prefix to the beginning of the property name, and do this once for each vendor prefix applicable. After declaring vendor prefixed versions of a property, declare the non-prefixed version. For example:

div{
  -moz-transition:10px;
  -webkit-transition:10px;
  transition:10px;
}

When deploying several vendor prefixes, use the non prefixed version of the property last, for forward-compatibility (when it is no longer an 'experimental' property, the prefixes will be dropped).

To stay up to date on the current status of prefixing with sites like Shouldiprefix.com

Enhanced Box Models

Several new properties are being introduced for styling box models.

  • border-radius: create rounded corners
  • box-shadow: cast shadow effect from box
  • border-image: use graphic for border styles (vendor prefixes required)
  • text-shadow: cast shadow effect from content
  • opacity: alter the transparency (alpha channel)
  • rgba() color values: define the color and transparency at the same time
border-radius

Box models can now be rendered with rounded corners, enabling elements that will display as circles, ovals, teardrops, etc.

border-radius requires a pixel amount determining the radius of the curved round corners, eg:

  • well supported. does not require vendor prefixes
  • border-radius:25px; all 4 corners rounded
  • border-radius:25px 0px; two corners rounded, two not rounded
  • from one to four space delimited values can be applied, one for each corner, clockwise from top left
  • supports the variants: border-bottom-left-radius, border-bottom-right-radius, border-top-left-radius, border-top-right-radius
  • you can even set the slant of the rounded corners
  • if you find that content can sometimes get too close to a rounded edge corner, consider increasing the padding
  • Border radius code generator
box-shadow

Makes a box model cast a shadow effect, with a result much like Photoshop's popular "Drop Shadow" layer style.

box-shadow requires space delimited pixel values to describe how far to cast the shadow horizontally, vertically:

  • well supported. does not require vendor prefixes
  • h-shadow (required) distance to horizontal cast - pixel value
  • v-shadow (required) distance to vertical cast - pixel value
  • blur (optional) blur distance - pixel value
  • spread (optional) size of shadow - pixel value
  • color (optional) color of shadow
  • Basic example: box-shadow:5px 5px 10px black;
  • Cast shadow inside the box: box-shadow:inset 0px 0px 10px 5px #000000;
  • Box shadow code generator
text-shadow

Creates a shadow effect on text, similar to how box-shadow works.

text-shadow requires space delimited pixel values to describe how far to cast the shadow horizontally, vertically:

  • well supported. does not require vendor prefixes
  • h-shadow (required) distance to horizontal cast - pixel value
  • v-shadow (required) distance to vertical cast - pixel value
  • blur (optional) blur distance - pixel value
  • spread (optional) size of shadow - pixel value
  • color (optional) color of shadow
  • For example: text-shadow:5px 5px 10px black;
  • Text shadow code generator
border-image

If CSS border styles are insufficient for your design goals, it is now possible to use images as well. A single image is 'sliced' into 9 sections, with each edge and corner segment used as the corresponding border edge/corner. The thickness of an image-based border will depend on the width and height of the images at the corners.

border-image properties

  • well supported. does not require vendor prefixes
  • border-image-source:
  • border-image-slice:
  • border-image-width:
  • border-image-outset:
  • border-image-repeat:
  • shorthand property: border-image: <source> <slice {1,4}> / <width {1,4}> <outset> <repeat{1,2}>;
  • Border Images code generator

Download and examine a border-image example.

An excellent tutorial on border-image is available from SitePoint

new color model

CSS3 provides another way to define the color of an element, this time allowing you to simultaneously assign the opacity as well as a RGB color value. The rgba() function assigns values to Red, Green, Blue and Alpha (opacity) channels. For example: 

.new_color_options{
	/* green and 75% opacity */
	background-color:rgba(0, 255, 0, 0.75);
}
.it_works_anywhere_you_use_color{
	/* blue and 10% opacity */
	color:rgba(0, 0, 255, 0.10);
}

opacity

Opacity affects the transparency of the selected element and descendents.

  • well supported. does not require vendor prefixes
  • Opacity code generator
.semi_transparent{
	/* 70% opacity */
	opacity: 0.70;
}
.completely_transparent{
	/* 0% opacity */
	opacity: 0;
}

Download and examine a demonstration of using opacity with backgrounds.

Transitions

Transitions allow changes to CSS properties to occur smoothly over a specified period, providing an animated effect. While not supported by all browsers, typical usage degrades gracefully. If the transition is a critical part of the user experience, javascript should be used instead of CSS3.

  • transition-property: the property involved in the transition.
  • transition-duration: how long the transition takes to complete (in seconds)
  • transition-timing-function: how the transition effect animates. Sample values include
    • ease
    • ease-in
    • ease-out
    • ease-in-out
    • linear
    • cubic-bezier (define your own timing curve)

Shorthand transitions

The basic 3 components of defining transitions can be declared in one line with the shorthand format (transition:property duration timing-function;), for example:

transition pattern: transition:property durationInSeconds timingFunction delay;

Example

Consider the following pseudoclassed anchor tag:

a{
	padding:5px 10px;
	background:#690;
	color:white;
	font:bold 12px arial;
}
a:hover{
	padding:5px 10px;
	background:#9c3;
}

This creates a standard hover effect on a background...

Hover Effect

Next we will add the transition code. Firefox requires the vendor prefix -moz-, microsoft uses -ms- and both Chrome, Opera and Safari use the -webkit- implementation. Anytime the state of the element undergoes a change, the transition will be triggered.

a{
	padding:5px 10px;
	background:#690;
	color:white;
	font:bold 12px arial;

	/* vendor specific transitions */
	-webkit-transition:background 0.3s ease;
	-moz-transition:background 0.3s ease;
	-ms-transition:background 0.3s ease;

	/* generic transitions */
	transition:background 0.3s ease;
}

Smooth Transition Hover Effect!

Test the new button in either Firefox, Safari or Chrome to see the transitions in action. test in IE or Opera to see the 'graceful degradation' of the CSS3 effect in non-supporting browsers.

Additional Transition Options

  • Delay the execution of the transition in seconds with the transition-delay property, eg:
    transition-delay:0.5s;
  • Apply the transition to a select number of properties with a comma delimited list, eg:
    transition:color 0.3s ease, background-color 0.1s linear;
  • Apply the transition to all properties that can be transitioned with the all value, eg:
    transition:all 0.3s ease;

NOTE: Not all CSS properties can animate as a transition. Consult the list of animate-able properties before using.

Learn more on transitions from the Mozilla Developer Network's Transitions Guide

Transforms

The transform property allows for the skewing, stretching, scaling and rotating of page elements.

  • Syntax: transform: <transform-function>;
  • Transform functions include: rotate() scale() skew() and translate()
  • One or more transform-functions may be applied. If there is more than one, they are applied from left to right
  • Note how elements surrounding a transformed element will not be affected by the transform (eg: transformed content may overlap other content)
  • Apply rotation with positive (clockwise) or negative (counter clockwise) values with deg units
    .rotate_clockwise{	
	transform:rotate(5deg);
	border-color:red;
}
.rotate_counter_clockwise{	
	transform:rotate(-15deg);
	border-color:orange;
}

  • Apply scaling with decimal values where 1 is default size:
    .scale_small{	
	transform:scale(.75);
	border-color:red;
}
.scale_big{	
	transform:scale(1.25);
	border-color:orange;
}

  • Skew with a comma delimited list of x and y coordinates as deg (degrees):
    .skew{	
	transform:skew(-5deg, 25deg);
	border-color:red;
}

  • The translate transform will move an element with comma delimited x, y coordinates:
    .translate{
	transform:translate(10px, -10px);
	border-color:red;
}

  • Apply multiple transformations with a space delimited list:
    .many_transformations{
	/* scale and skew */	
	transform:scale(.75) skew(-5deg, 10deg);
	border-color:red;
}

  • You can combine transforms with other CSS techniques, such as transitions, allowing for interesting interactive design options. eg: try hovering over the images below...

Animations and Keyframes

CSS animations allow for more sophisticated transitions. The developer first defines various stages of the transition, then a CSS rule is defined that will 'trigger' the animation.

animation pattern animation: name duration iterationCount delay

A keyframe allows you to define the intermediate steps in an animation sequence by defining waypoints (keyframes).

  1. first, define the various stages of the animation. Choose your own identifier, and define at minimum, how the animation will begin (eg: 0%), and end (eg: 100%).
    /* define 3 waypoints for an animation named 'identifier' */
@keyframes identifier{
	0%{		box-shadow:2px 2px 5px rgba(0,0,255, .2);	 	}
	50%{	box-shadow:2px 2px 5px rgba(0,0,255, .9);	 	}
	100%{	box-shadow:2px 2px 5px rgba(0,0,255, .2);	 	}
}
  2. then reference the identifier in a CSS animation
    /* run the animation */
input:focus{
	animation:identifier 1.5s infinite ease-in-out;
}

Defining Iterations

Use the iteration count to deterine how many times an animation should repeat

  • infinite loop animation:identifier 1.5s infinite;
  • run only once: animation:identifier 1.5s 1;
  • loop 3 times: animation:identifier 1.5s 3;
  • loop 2 and a half times: animation:identifier 1.5s 2.5;

Browser Support

Animations are supported with vendor prefixes for most up to date major browsers. Note: animations are still in experimental stage (test animations in different browsers to know if they will work)

  • Safari (uses prefix -webkit-)
  • Chrome (uses prefix -webkit-)
  • Firefox (uses prefix -moz-)
  • IE (uses prefix -ms-)
/* note the placement of vendor prefixes for keyframes... */
@-webkit-keyframes identifier{
	0%{ ... }	50%{ ... }	100%{ ... }
}
@-moz-keyframes identifier{
	0%{ ... }	50%{ ... }	100%{ ... }
}
@-ms-keyframes identifier{
	0%{ ... }	50%{ ... }	100%{ ... }
}
@keyframes identifier{
	0%{ ... }	50%{ ... }	100%{ ... }
}
/* also use prefixes when running the animation */
div:hover{
	-webkit-animation: identifier 2s 1; 	
	-moz-animation: identifier 2s 1;
	-ms-animation: identifier 2s 1;
	animation: identifier 2s 1;
}

CSS3 Functions

CSS3 functions provide options for creating more 'dynamic' code

  • attr() can be used to insert the textual contents of a tag's attribute. Note: it can only be used with content: property, usually in conjunction with the :before or :after pseudoclass 
    /* display the href of an anchor tag */
a:after{
	content: attr( href );
}
/* display the alt of an image tag */
img:before{
	content: attr(  alt );
}
  • calc() ideal for using mixed unit lengths (eg: percentages and pixels)
    property:calc(expression) the expression may include mather opertators + - * / and round brackets may be used to nest calculations and imply order of operations. NOTE: Math operators must have spaces before and after. Examples: 
    /* mix fixed and variable widths */
#fixed_width{		width:100px;			 }
#variable_width{	width:calc(100% - 100px);}

/* make 5 even width columns */
width:calc( 100% / 5);

/* make 4 even width columns with an extra 40px space remaining */
width:calc( 100% / 4 - 40px);
  • Examine a page demonstrating some uses of calc()

Backgrounds

Background Gradient

Instead of requiring images for gradients, CSS3 can now handle the job. At present, vendor prefixes are required in all cases. IE is expected to support the feature in version 10. The basic pattern is:
background: -vendorprefix-linear-gradient(starting-edge, starting-color, end-color); 

#gradient{
	height:30px;
	width:100px;
	border:solid 1px black;
	background: -moz-linear-gradient(top, white, cornflowerblue);
	background: -webkit-linear-gradient(top, white, cornflowerblue);
	background: -o-linear-gradient(top, white, cornflowerblue);
	background: linear-gradient(top, white, cornflowerblue);
}

There will be plenty of gradient variations avaiable, though only linear-gradient has good support at present.

Css-tricks provides detailed examples for a variety of gradient useages, and experiment with the CSS gradient code generator

Multiple Backgrounds

CSS3 also provides for multiple backgrounds/background images. If overlapping backgrounds include transparent sections the background layers beneath will show through.

You still deploy the background property as before, but you can now provide a comma delimited list of values to it:

#multiple_backgrounds{
	border:solid 1px black;
	height:78px;
	background:	url("images/cloud_small.png") repeat-x 60% 0,
			url("images/cloud.png") repeat-x 30% 0,
			url("images/sun.png") no-repeat,
			cornflowerblue;
}
